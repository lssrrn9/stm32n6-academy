Leccion 1: Introduccion al STM32N6

El STM32N6 es el microcontrolador mas avanzado de la familia STM32 de STMicroelectronics. Disenado para aplicaciones industriales de proxima generacion, integra por primera vez un procesador neuronal (NPU) junto a un nucleo CPU de alto rendimiento.

Arquitectura heterogenea

El STM32N6 combina dos procesadores especializados en un solo chip:

  • Cortex-M55 — CPU principal de 32 bits, arquitectura ARMv8.1-M. Ejecuta el firmware principal, protocolos de comunicacion y logica de control.
  • Ethos-U55 — NPU (Neural Processing Unit) dedicado para acelerar inferencias de IA/ML. Soporta operaciones INT8 y INT16 con hasta 512 MAC/ciclo.

Concepto clave: La arquitectura heterogenea permite ejecutar inferencias de IA localmente (edge AI) sin depender de la nube, reduciendo latencia y consumo de energia.

Leccion 2: Instalacion de STM32CubeIDE

STM32CubeIDE es el entorno de desarrollo integrado oficial de STMicroelectronics. Combina editor de codigo, compilador GCC, depurador GDB y el configurador grafico del microcontrolador en una sola herramienta gratuita.

Paso 1: Descarga

  1. Visita www.st.com/stm32cubeide
  2. Haz clic en "Get Software" y selecciona tu sistema operativo (Windows, Linux o macOS).
  3. Registrate con una cuenta gratuita de myST si es necesario.
  4. Descarga el instalador (aproximadamente 1.5 GB).

Paso 2: Instalacion

  1. Ejecuta el instalador descargado con permisos de administrador.
  2. Acepta los terminos de la licencia.
  3. Selecciona el directorio de instalacion (por defecto: C:\ST\STM32CubeIDE).
  4. Elige los componentes: asegurate de incluir STM32CubeMX integrado.
  5. Espera la instalacion (puede tardar 15-30 minutos).

Paso 3: Configuracion del Workspace

  1. Al iniciar STM32CubeIDE, selecciona un directorio de workspace.
  2. Recomendacion: C:\STM32_Workspace\N6_Projects
  3. El workspace almacena todos tus proyectos y configuraciones.

Paso 4: Crear el primer proyecto

Menu: File → New → STM32 Project

  1. Se abrira el MCU Selector (STM32CubeMX integrado).
  2. En el campo de busqueda, escribe STM32N6 y selecciona tu variante (ej: STM32N657xx).
  3. Haz clic en Next.
  4. Asigna un nombre al proyecto: N6_Blink_LED
  5. Selecciona C como lenguaje y haz clic en Finish.

Estructura de carpetas generada

N6_Blink_LED/ | +-- Core/ | +-- Inc/ <-- Headers (.h) | | +-- main.h | | | +-- Src/ <-- Codigo fuente (.c) | +-- main.c <-- Archivo principal | +-- Drivers/ | +-- STM32N6xx_HAL_Driver/ <-- Libreria HAL | +-- CMSIS/ <-- CMSIS-Core | +-- STM32N6xx_FLASH.ld <-- Script de linker +-- startup_stm32n6xx.s <-- Codigo de arranque +-- .cproject <-- Configuracion del proyecto

Consejo: La primera compilacion puede tardar varios minutos porque STM32CubeIDE debe compilar toda la libreria HAL. Las siguientes compilaciones seran mucho mas rapidas.

Leccion 3: Primer proyecto HAL — Blink LED en PC13

En esta leccion escribiremos el firmware mas basico para el STM32N6: un programa que hace parpadear un LED conectado al pin PC13. Este es el equivalente al "Hola Mundo" en el mundo de los microcontroladores.

Conceptos previos

  • HAL (Hardware Abstraction Layer): libreria proporcionada por ST que abstrae el acceso a los perifericos del microcontrolador.
  • GPIO (General Purpose Input/Output): pines configurables como entrada o salida digital.
  • PC13: pin 13 del puerto C. En la mayoria de placas de desarrollo STM32, este pin controla un LED integrado.

Codigo HAL completo comentado linea por linea

// ============================================================================
// main.c - Blink LED en PC13 para STM32N6
// ============================================================================

#include "stm32n6xx_hal.h"   // Header HAL para STM32N6: funciones de
                             // inicializacion y control de perifericos

// ----------------------------------------------------------------------------
// Prototipos de funciones
// ----------------------------------------------------------------------------
void SystemClock_Config(void);   // Configura el arbol de relojes del MCU
static void MX_GPIO_Init(void);  // Configura los pines GPIO

// ----------------------------------------------------------------------------
// Funcion principal: punto de entrada del programa
// ----------------------------------------------------------------------------
int main(void)
{
    // 1. Inicializar el sistema HAL: debe ser la primera instruccion
    //    Resetea todos los perifericos, configura la memoria cache y
    //    prepara el sistema para su uso
    HAL_Init();

    // 2. Configurar el arbol de relojes del sistema
    //    Establece las frecuencias de trabajo de CPU, buses y perifericos
    SystemClock_Config();

    // 3. Inicializar los pines GPIO
    //    Configura PC13 como salida digital
    MX_GPIO_Init();

    // ----------------------------------------------------------------------------
    // Bucle principal (super-loop): se ejecuta indefinidamente
    // ----------------------------------------------------------------------------
    while (1)
    {
        // Invertir el estado logico del pin PC13 (ON -> OFF, OFF -> ON)
        // Primer parametro: puerto GPIO (GPIOC)
        // Segundo parametro: numero de pin (GPIO_PIN_13)
        HAL_GPIO_TogglePin(GPIOC, GPIO_PIN_13);

        // Espera bloqueante de 500 milisegundos
        // El CPU se queda en modo espera durante este tiempo
        HAL_Delay(500);

        // Efecto visual: el LED parpadea a 1 Hz (encendido 500ms, apagado 500ms)
    }
}

// ----------------------------------------------------------------------------
// Configuracion del arbol de relojes
// ----------------------------------------------------------------------------
void SystemClock_Config(void)
{
    // Estructura de configuracion del PLL (Phase Locked Loop)
    RCC_OscInitTypeDef RCC_OscInitStruct = {0};
    RCC_ClkInitTypeDef RCC_ClkInitStruct = {0};

    // --- Configuracion de osciladores ---
    RCC_OscInitStruct.OscillatorType = RCC_OSCILLATORTYPE_HSI;
    // Seleccionamos el HSI (High Speed Internal) como fuente de reloj

    RCC_OscInitStruct.HSIState = RCC_HSI_ON;
    // Activamos el oscilador interno HSI (16 MHz por defecto)

    RCC_OscInitStruct.HSICalibrationValue = RCC_HSICALIBRATION_DEFAULT;
    // Usamos el valor de calibracion de fabrica para precision

    RCC_OscInitStruct.PLL.PLLState = RCC_PLL_ON;
    // Activamos el PLL para multiplicar la frecuencia

    RCC_OscInitStruct.PLL.PLLSource = RCC_PLLSOURCE_HSI;
    // El PLL usa el HSI como fuente de entrada

    RCC_OscInitStruct.PLL.PLLM = 4;    // Division pre-PLL:  16 MHz / 4 = 4 MHz
    RCC_OscInitStruct.PLL.PLLN = 60;   // Multiplicacion:     4 MHz * 60 = 240 MHz
    RCC_OscInitStruct.PLL.PLLP = 2;    // Division post-PLL: 240 MHz / 2 = 120 MHz (SYSCLK)

    // Aplicamos la configuracion de osciladores
    HAL_RCC_OscConfig(&RCC_OscInitStruct);

    // --- Configuracion de buses ---
    RCC_ClkInitStruct.ClockType = RCC_CLOCKTYPE_HCLK | RCC_CLOCKTYPE_SYSCLK
                                 | RCC_CLOCKTYPE_PCLK1 | RCC_CLOCKTYPE_PCLK2;
    // Configuramos todos los tipos de reloj

    RCC_ClkInitStruct.SYSCLKSource = RCC_SYSCLKSOURCE_PLLCLK;
    // SYSCLK proviene del PLL (120 MHz)

    RCC_ClkInitStruct.AHBCLKDivider = RCC_SYSCLK_DIV1;
    // AHB (Advanced High-performance Bus): sin division -> 120 MHz

    RCC_ClkInitStruct.APB1CLKDivider = RCC_HCLK_DIV2;
    // APB1 (Advanced Peripheral Bus 1):   120 MHz / 2 = 60 MHz

    RCC_ClkInitStruct.APB2CLKDivider = RCC_HCLK_DIV2;
    // APB2 (Advanced Peripheral Bus 2):   120 MHz / 2 = 60 MHz

    // Aplicamos la configuracion de relojes
    HAL_RCC_ClockConfig(&RCC_ClkInitStruct, FLASH_LATENCY_2);
    // FLASH_LATENCY_2: 2 ciclos de espera para acceso a Flash a 120 MHz
}

// ----------------------------------------------------------------------------
// Inicializacion de GPIO
// ----------------------------------------------------------------------------
static void MX_GPIO_Init(void)
{
    // Variable de configuracion del GPIO
    GPIO_InitTypeDef GPIO_InitStruct = {0};

    // --- Paso 1: Habilitar el reloj del puerto GPIOC ---
    // Todos los perifericos necesitan su reloj habilitado antes de usarse
    __HAL_RCC_GPIOC_CLK_ENABLE();

    // --- Paso 2: Configurar el pin PC13 ---
    GPIO_InitStruct.Pin = GPIO_PIN_13;           // Pin 13 del puerto C
    GPIO_InitStruct.Mode = GPIO_MODE_OUTPUT_PP;  // Modo: Salida Push-Pull
    GPIO_InitStruct.Pull = GPIO_NOPULL;          // Sin resistencia pull-up/down
    GPIO_InitStruct.Speed = GPIO_SPEED_FREQ_LOW; // Velocidad baja (suficiente para LED)

    // Aplicamos la configuracion
    HAL_GPIO_Init(GPIOC, &GPIO_InitStruct);

    // --- Paso 3: Estado inicial del pin ---
    // Apagamos el LED al inicio (logica negativa en muchas placas)
    HAL_GPIO_WritePin(GPIOC, GPIO_PIN_13, GPIO_PIN_SET);
}

// ----------------------------------------------------------------------------
// Manejador de errores (callback por defecto de HAL)
// ----------------------------------------------------------------------------
void Error_Handler(void)
{
    // Si ocurre un error critico, entramos en bucle infinito
    // En produccion, aqui se reiniciaria el sistema o se activaria un fallo seguro
    __disable_irq();   // Desactivamos todas las interrupciones
    while (1) { }      // Bucle infinito de seguridad
}

Explicacion de cada funcion HAL utilizada

Funcion HALPropositoParametros clave

HAL_GPIO_Init()Configura uno o mas pines de un puerto GPIO segun la estructura GPIO_InitTypeDef.GPIO_TypeDef *GPIOx, GPIO_InitTypeDef *GPIO_Init
HAL_GPIO_TogglePin()Invierte el estado logico del pin especificado (HIGH -> LOW, LOW -> HIGH).GPIO_TypeDef *GPIOx, uint16_t GPIO_Pin
HAL_GPIO_WritePin()Escribe un valor logico explicito en el pin (GPIO_PIN_SET o GPIO_PIN_RESET).GPIO_TypeDef *GPIOx, uint16_t GPIO_Pin, GPIO_PinState PinState

Nota importante: HAL_Delay() es bloqueante: el CPU no hace nada util durante la espera. En aplicaciones reales se usan temporizadores por interrupcion (TIM) o un RTOS para multitarea.

Leccion 4: Arbol de relojes (Clock Tree)

El arbol de relojes (Clock Tree) es uno de los sistemas mas criticos de un microcontrolador. Determina la frecuencia de operacion de la CPU, los buses de comunicacion y todos los perifericos. Comprenderlo es esencial para optimizar el rendimiento y el consumo energetico.

Fuentes de reloj principales

  • HSI (High Speed Internal) — Oscilador RC interno de 16 MHz. Siempre disponible, no requiere componentes externos. Precision tipica: +/- 1% a 25 C.
  • HSE (High Speed External) — Cristal o resonador externo de 4-50 MHz. Mayor precision y estabilidad. Requiere cristal + condensadores externos.
  • PLL (Phase Locked Loop) — Circuito que multiplica/divide la frecuencia de entrada para generar frecuencias mayores.

Regla practica: Usa HSI para desarrollo rapido y prototipos. Usa HSE cuando necesites precision (comunicacion USB, Ethernet, audio) o el maximo rendimiento.

Diagrama del arbol de relojes

+-----------------------------------------------------------------------+ | STM32N6 Clock Tree (simplificado) | +-----------------------------------------------------------------------+ | | | +-----------+ +-----------+ | | | HSI | | HSE | | | | 16 MHz | | 8-50 MHz | | | | (interno) | |(cristal) | | | +-----+-----+ +-----+-----+ | | | | | | +----------+----------+ | | | | | +-----v-----+ | | | PLL | PLLM: division previa | | | | PLLN: multiplicacion | | | f_out = | PLLP: division posterior | | | f_in*N/M/P| | | +-----+-----+ | | | | | +-----v-----+ SYSCLK (hasta 400 MHz) | | | SYSCLK | | | +-----+-----+ | | | | | +-----v-----+ AHB Prescaler | | | HCLK | (1, 2, 4, 8, 16, 64, 128, 256, 512) | | | (AHB Bus) | | | +-----+-----+ | | | | | +----------+----------+ | | | | | | +----v----+ +-----v-----+ | | | APB1 | | APB2 | | | |PCLK1 | | PCLK2 | | | | max 200 | | max 200 | | | | MHz | | MHz | | | +----+----+ +-----+-----+ | | | | | | +----v----+ +-----v-----+ | | |TIM2,3,4 | |TIM1,15,16 | | | |I2C1,2 | | SPI1,4 | | | |USART2,3 | |USART1,6 | | | | SPI2,3 | | ADC1 | | | +---------+ +----------+ | | | +-----------------------------------------------------------------------+

Como configurar el Clock en STM32CubeIDE

  1. Abre el archivo .ioc (nombre del proyecto + .ioc) en la raiz del proyecto.
  2. Se abrira la interfaz grafica de STM32CubeMX.
  3. Navega a la pestana "Clock Configuration".
  4. En el diagrama grafico, selecciona la fuente de reloj (HSI o HSE) en el multiplexor de entrada.
  5. Configura los valores del PLL: PLLM, PLLN, PLLP.
  6. Los valores de los divisores AHB, APB1 y APB2 se ajustan automaticamente o manualmente segun tus necesidades.
  7. Presiona Ctrl+S para guardar. El codigo se regenera automaticamente.

Leccion 5: Configuracion de GPIO — Entrada y Salida

Los GPIO (General Purpose Input/Output) son los pines de proposito general del microcontrolador. Pueden configurarse como entradas para leer sensores y botones, o como salidas para controlar LEDs, reles y otros actuadores. Dominar la configuracion GPIO es fundamental para cualquier proyecto embebido.

Estructura GPIO_InitTypeDef explicada campo por campo

// Estructura de configuracion GPIO (definida en stm32n6xx_hal_gpio.h)
typedef struct {
    uint32_t Pin;        // Pin(es) a configurar: GPIO_PIN_0 ... GPIO_PIN_15
    uint32_t Mode;       // Modo de funcionamiento (ver tabla abajo)
    uint32_t Pull;       // Resistencia pull-up/pull-down
    uint32_t Speed;      // Velocidad de conmutacion del pin
    uint32_t Alternate;  // Funcion alternativa (AF) para perifericos
} GPIO_InitTypeDef;

Ejemplo practico: Boton + LED

Este proyecto controla un LED con un boton: el LED se enciende mientras se presiona el boton. El boton esta conectado a PA0 y el LED a PC13.

Conexion fisica tipica: Boton entre PA0 y GND (con pull-up interno). LED en PC13 (logica negativa: LOW = encendido).

// ============================================================================
// main.c - Boton en PA0 + LED en PC13 para STM32N6
// ============================================================================

#include "stm32n6xx_hal.h"

// Definiciones para claridad del codigo
#define BOTON_PIN    GPIO_PIN_0
#define BOTON_PORT   GPIOA
#define LED_PIN      GPIO_PIN_13
#define LED_PORT     GPIOC

void SystemClock_Config(void);
static void MX_GPIO_Init(void);

int main(void)
{
    HAL_Init();
    SystemClock_Config();
    MX_GPIO_Init();

    while (1)
    {
        // Leer el estado del boton en PA0
        // HAL_GPIO_ReadPin devuelve GPIO_PIN_SET (1) o GPIO_PIN_RESET (0)
        GPIO_PinState boton_presionado = HAL_GPIO_ReadPin(BOTON_PORT, BOTON_PIN);

        // El boton tiene pull-up: cuando NO se presiona -> HIGH (1)
        //                           cuando se presiona  -> LOW  (0)
        if (boton_presionado == GPIO_PIN_RESET)
        {
            // Boton presionado -> Encender LED (LOW activo)
            HAL_GPIO_WritePin(LED_PORT, LED_PIN, GPIO_PIN_RESET);
        }
        else
        {
            // Boton liberado -> Apagar LED
            HAL_GPIO_WritePin(LED_PORT, LED_PIN, GPIO_PIN_SET);
        }

        // Pequeno retardo para eliminar rebote del boton (debouncing)
        HAL_Delay(10);   // 10 ms es suficiente para la mayoria de botones
    }
}

static void MX_GPIO_Init(void)
{
    GPIO_InitTypeDef GPIO_InitStruct = {0};

    // === Habilitar relojes de los puertos que usaremos ===
    __HAL_RCC_GPIOA_CLK_ENABLE();   // Puerto A (boton)
    __HAL_RCC_GPIOC_CLK_ENABLE();   // Puerto C (LED)

    // === Configurar PA0 como entrada con pull-up ===
    GPIO_InitStruct.Pin  = BOTON_PIN;              // Pin 0
    GPIO_InitStruct.Mode = GPIO_MODE_INPUT;        // Modo entrada
    GPIO_InitStruct.Pull = GPIO_PULLUP;            // Pull-up interno
    HAL_GPIO_Init(BOTON_PORT, &GPIO_InitStruct);   // Aplicar en GPIOA

    // === Configurar PC13 como salida push-pull ===
    GPIO_InitStruct.Pin   = LED_PIN;                // Pin 13
    GPIO_InitStruct.Mode  = GPIO_MODE_OUTPUT_PP;    // Salida push-pull
    GPIO_InitStruct.Pull  = GPIO_NOPULL;            // Sin pull
    GPIO_InitStruct.Speed = GPIO_SPEED_FREQ_LOW;    // Velocidad baja
    HAL_GPIO_Init(LED_PORT, &GPIO_InitStruct);      // Aplicar en GPIOC

    // Estado inicial: LED apagado
    HAL_GPIO_WritePin(LED_PORT, LED_PIN, GPIO_PIN_SET);
}

void SystemClock_Config(void)
{
    // Configuracion basica de reloj (HSI + PLL para 120 MHz)
    RCC_OscInitTypeDef RCC_OscInitStruct = {0};
    RCC_ClkInitTypeDef RCC_ClkInitStruct = {0};

    RCC_OscInitStruct.OscillatorType = RCC_OSCILLATORTYPE_HSI;
    RCC_OscInitStruct.HSIState = RCC_HSI_ON;
    RCC_OscInitStruct.HSICalibrationValue = RCC_HSICALIBRATION_DEFAULT;
    RCC_OscInitStruct.PLL.PLLState = RCC_PLL_ON;
    RCC_OscInitStruct.PLL.PLLSource = RCC_PLLSOURCE_HSI;
    RCC_OscInitStruct.PLL.PLLM = 4;
    RCC_OscInitStruct.PLL.PLLN = 60;
    RCC_OscInitStruct.PLL.PLLP = 2;
    HAL_RCC_OscConfig(&RCC_OscInitStruct);

    RCC_ClkInitStruct.ClockType = RCC_CLOCKTYPE_HCLK | RCC_CLOCKTYPE_SYSCLK
                                | RCC_CLOCKTYPE_PCLK1 | RCC_CLOCKTYPE_PCLK2;
    RCC_ClkInitStruct.SYSCLKSource = RCC_SYSCLKSOURCE_PLLCLK;
    RCC_ClkInitStruct.AHBCLKDivider = RCC_SYSCLK_DIV1;
    RCC_ClkInitStruct.APB1CLKDivider = RCC_HCLK_DIV2;
    RCC_ClkInitStruct.APB2CLKDivider = RCC_HCLK_DIV2;
    HAL_RCC_ClockConfig(&RCC_ClkInitStruct, FLASH_LATENCY_2);
}

void Error_Handler(void)
{
    __disable_irq();
    while (1) { }
}

Evaluacion de conocimientos

Responde las siguientes preguntas para verificar tu comprension del modulo.

Pregunta 1: Arquitectura del STM32N6

Que componente del STM32N6 se encarga de acelerar las inferencias de inteligencia artificial?

Correcto! El Ethos-U55 es el Neural Processing Unit (NPU) integrado en el STM32N6. Es un coprocesador especializado que acelera operaciones de redes neuronales (hasta 512 MAC/ciclo), liberando al Cortex-M55 de esta carga computacional. El Cortex-M55 es la CPU principal que ejecuta el firmware, pero no es su funcion acelerar IA.

Pregunta 2: Fuentes de reloj

Cual es la frecuencia tipica del oscilador HSI (High Speed Internal) del STM32N6?

Correcto! El HSI (High Speed Internal) del STM32N6 oscila a 16 MHz por defecto. Es un oscilador RC interno que siempre esta disponible despues del reset, lo que permite iniciar el microcontrolador sin componentes externos. Si se necesita mayor precision, se puede usar un cristal HSE externo o calibrar el HSI via software.

Pregunta 3: Funciones HAL para GPIO

Que funcion HAL se utiliza para invertir (toggle) el estado logico de un pin GPIO?

Correcto! HAL_GPIO_TogglePin() invierte el estado actual del pin: si esta en HIGH pasa a LOW, y viceversa. Es la funcion ideal para hacer parpadear un LED sin necesidad de llevar un estado manual. HAL_GPIO_WritePin() requiere especificar el estado deseado, HAL_GPIO_ReadPin() solo lee, y HAL_GPIO_Init() configura el pin pero no cambia su estado logico.
